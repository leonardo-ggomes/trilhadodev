Processo de Debug

O que é Debug?

É o processo de identificar, analisar e remover bugs (falhas) do seu código.

Mais do que apenas consertar erros, debugar é uma investigação para entender por que o programa não está se comportando como esperado.

Como Investigar?

Existem várias formas de encontrar o problema:

  • Console Log: Imprimir valores para ver o que acontece em tempo real.
  • Breakpoints: Pausar o código em uma linha específica.
  • Step-by-Step: Avançar linha por linha na execução.

Ferramentas de IDE

IDEs modernas como o VS Code possuem painéis de debug que mostram:

Variables: O valor atual de cada variável.

Call Stack: A ordem das funções que foram chamadas.

Watch: Monitoramento de expressões específicas.

Visualizando o Erro

O ponto vermelho (breakpoint) indica onde a execução vai parar:


● 1 function calcularTotal(preco, qtd) {
   2    const imposto = 0.15;
   3    // Debugger para aqui na linha 1
   4    return (preco * qtd) * imposto;
   5 }

Debugar antes da produção evita "incêndios" no sistema final!
