Arquitetura Cliente-Servidor

O Fluxo de Dados

A arquitetura funciona em um ciclo de Request/Response. O cliente inicia a conversa e o servidor aguarda passivamente por pedidos.

Metáfora do Restaurante

Cliente: O Browser (Chrome/Safari).

Requisição: O endereço (URL) que você digita.

Servidor: O computador remoto que guarda o site.

Por que usar?

  • Segurança: Dados sensíveis não ficam no dispositivo do usuário.
  • Consistência: Se o preço de um produto muda no servidor, muda para todos os clientes instantaneamente.

A Comunicação (HTTP)

O servidor responde com códigos de status para dizer o que aconteceu:

✅ 200: OK! Deu tudo certo.

⚠️ 404: Not Found! O recurso não existe.

🚫 500: Server Error! O servidor "quebrou" ao processar.

Conceito de Stateless

A maioria das conexões web são Stateless (sem estado). Isso significa que cada requisição é nova e isolada.

O servidor não "lembra" quem você é apenas pela conexão física. Para ele saber que você está logado, você precisa enviar um Token ou Cookie em cada pedido.

Variedades de Servidores

Um servidor não é apenas para sites. Existem diversos tipos:

DB Server: Armazena e organiza dados.

Web Server: Entrega HTML, CSS e JS.

API Server: Entrega dados brutos (JSON).
