Arquitetura MVC

O que é MVC?

O padrão Model-View-Controller é uma forma de organizar seu código dividindo-o em três camadas independentes.

Pense nisso como a cozinha de um restaurante: cada um tem seu papel definido para que o prato chegue perfeito à mesa.

1. Model (O Cérebro)

O Model cuida dos dados e da lógica de negócios. Ele sabe como salvar um usuário no banco de dados ou como calcular o preço total de um carrinho.

Exemplo: usuarioModel.js - Contém o esquema da tabela e as regras de validação.

2. View (A Cara)

A View é o que o usuário vê. São os arquivos HTML, CSS e as respostas visuais do sistema.

Exemplo: perfil.html - Mostra os dados que o Model forneceu, mas formatados para o humano ler.

3. Controller (O Garçom)

O Controller é o intermediário. Ele recebe o clique do usuário na View, pede a informação para o Model e devolve a resposta para a View.

Exemplo: userController.js - Recebe a rota /login e decide o que fazer.

Estrutura no Node.js

Veja como seu projeto deve ficar organizado visualmente:

  • src/
    • models/ (Lógica de Dados)
    • views/ (Interfaces)
    • controllers/ (Regras de Fluxo)

Por que usar?

  • Manutenção: É fácil achar erros.
  • Escalabilidade: O projeto pode crescer sem virar bagunça.
  • Equipe: Um dev mexe no CSS enquanto outro mexe no Banco.
