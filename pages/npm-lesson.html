Dominando o NPM

O que é o NPM?

O Node Package Manager é como uma enorme biblioteca virtual. Ele permite que você baixe ferramentas prontas (pacotes) criadas por outros desenvolvedores para usar no seu projeto.

"Não reinvente a roda, use o NPM para acelerar seu desenvolvimento."

1. Iniciar o Projeto

Tudo começa com o comando npm init. Ele cria o arquivo package.json.

O package.json funciona como o RG do seu projeto: ele guarda o nome, a versão e a lista de tudo que o projeto precisa para rodar.

2. Instalar Pacotes

Para baixar uma biblioteca (como uma para manipular datas ou banco de dados), usamos:

npm install nome-do-pacote

Isso cria a pasta node_modules, onde os arquivos reais da biblioteca ficam guardados.

3. Compartilhar e Gerenciar

Você nunca envia a pasta node_modules para outros (ela é muito pesada!).

Basta enviar o package.json. Quando outra pessoa baixar seu projeto, ela só precisa digitar npm install para recuperar todas as bibliotecas automaticamente.

Resumo da Ópera

  • NPM: Loja de ferramentas.
  • package.json: Lista de compras.
  • node_modules: Onde as ferramentas ficam.
